# Distribution Checklist

Sequenced posting plan for the author. Designed for execution in small bite-sized sessions, total ~2-3 hours of active work spread over 1-2 weeks.

## Pre-Launch Setup (~60 min)

- [ ] **Create the dedicated Gmail account** `empirical.homiletics@gmail.com` (or a fallback like `openempiricalhomiletics@gmail.com` if that's taken). Set auto-forwarding to your personal email. This is the one ongoing maintenance task.
- [ ] **Register a GitHub organization or user repo** — suggested name: `open-empirical-homiletics`. Make public. Add the full `2026-04-17-sermon-calibration-design.md` as `README.md` at root, plus the supporting files from this directory under `docs/`. Commit the CC BY 4.0 license file (`LICENSE`) at root.
- [ ] **Verify Pleizier's current email** on [pthu.nl](https://www.pthu.nl/en/about-us/people/t.t.j.pleizier/) before sending. Email addresses change.
- [ ] **Read the full draft once** and flag anything overclaiming or inaccurate. This is your last chance before it has a DOI.

## Launch Day (~60 min of active time)

- [ ] **Submit to SocArXiv**: https://osf.io/preprints/socarxiv — use `socarxiv-abstract.md` for the abstract and metadata; upload the design doc as the PDF/markdown content. DOI is minted automatically within ~24 hours. Free.
- [ ] **Push the GitHub repo** with README, LICENSE, docs folder. Confirm the repository URL is accessible.
- [ ] **Update the design doc and supporting files** to replace `[SocArXiv DOI]` and `[repository URL]` placeholders with the real links. Commit. Push.
- [ ] **Send the Pleizier email** (`email-to-pleizier.md`). This is the single highest-leverage action in the whole sequence. Do not delay waiting for a "perfect moment" — send it the same day you publish.

## Week 1 Follow-up (~30 min)

- [ ] **Cross-post to ResearchGate** as a project (not a preprint — ResearchGate has a "Project" feature). Link to the SocArXiv DOI.
- [ ] **Post to r/Reformed and r/Christianity** with a short personal note + link to the preprint. Not r/AcademicBiblical (wrong subject area).
- [ ] **If you are a member of the Academy of Homiletics listserv**, forward `listserv-post-academy.md` with a personal intro line. If not, ask a member of the guild who read the preprint and found it credible to forward on your behalf.
- [ ] **Tweet/Bluesky**: two-line summary + link. Tag relevant accounts (Psychological Science Accelerator, ManyBabies, Center for Open Science, OSF).

## Weeks 2-4 Optional Amplification (~15 min each)

- [ ] **Submit a Short Research Note to *Homiletic*** (open access, Vanderbilt-hosted). Summarize the invitation in 500-800 words; point to the preprint for detail.
- [ ] **Email Tier 2 named candidates** from the design doc (Alcántara/Kim, Thomas, Yang, Kaufman) — one short personal note each with link to the preprint. Do NOT batch-blast; personalize.
- [ ] **Post to one or two Reformed-preaching Facebook groups** if you are already a member ("Pastors Preaching Well," "Reformed Pastors," etc.). Skip if not already a member — cold posts rarely convert.

## After 30 Days: Signal Test

Measure: how many qualified responses arrived via `empirical.homiletics@gmail.com` + GitHub issues?

- **Fewer than 5 qualified responses**: the latent demand is not there. Let it sit. The preprint and GitHub repo remain discoverable forever; future researchers can find them.
- **5-15 qualified responses**: promising. Reply to each personally. Ask if any of them are willing to take ownership. If someone volunteers, begin handoff conversation.
- **15+ qualified responses, or any response from Pleizier or a Tier 1 contact**: the project has a path. Reach out individually to the most credentialed/motivated and explore who could serve as founding methodologist co-lead.

## Long-term Maintenance

**Near zero.** The SocArXiv preprint and GitHub repo are permanent. Your only ongoing commitments:

- **Respond to emails and issues** as they arrive. If someone asks "can I adopt this?", say yes and help with handoff.
- **Quarterly check-in** (~15 min): glance at email and GitHub issues. Delete spam, forward serious inquiries to the adopter if one has emerged.
- **If no adopter appears in 12 months**: archive the repository with a note indicating no adoption occurred, but the preprint remains available. That is itself a form of intellectual honesty.

## What Not To Do

- **Don't build more infrastructure.** No rater app. No prototype code. No fancy website. Infrastructure without adoption is the failure mode codex warned about (*ADHD founder-energy as avoidance of the hard recruitment work*).
- **Don't chase metrics.** Downloads, GitHub stars, ResearchGate views are vanity metrics. The only thing that matters is: does a qualified person adopt, within the next 12 months?
- **Don't soft-pedal the constraint.** If someone reaches out and says "can we collaborate?", your answer is: "I can offer this design and answer questions. I cannot lead or commit ongoing time. If you want to lead, please take it."
